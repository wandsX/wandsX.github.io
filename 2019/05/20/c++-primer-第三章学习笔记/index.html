本文初识标准库和迭代器,以及using在 c++11中新用处。

0x00. 命名空间的using 声明

头文件不应该使用using 声明,因为头文件的内容会拷贝到所有引用它的文件中。这样容易产生名字冲突。

0x01. 标准库类型string

1.1 初始化

string s1;          // 默认初始化 空串

string s2(s1);      // s2是s1的副本。

string s2 = s1;     // 等价于s2(s1)。

string s3("value"); // 等价写法 string s3 = "value";

string s4(n, 'c');  // s4为连续n个字符c组成的字符串。

1.2 直接初始化和拷贝初始化

如果使用等号初始化一个变量的时候, 实际上执行的是拷贝初始化，反之则是直接初始化。

当初始值只有一个时，使用直接初始化或靠被初始化都行。

string s5 = "hiya";    //拷贝初始化。
string s6("jojo");    // 直接初始化。
string s7(10, 'c');    // 直接初始化，s7的内容是cccccccccc

对于多个值进行初始化的时候，用拷贝初始化的方式来处理也不是不可以,不过需要显式地的创建一个(临时)对象用于拷贝：

string s8 = string(10, 'c');

// 上面这条语句等价于下面的
string temp(10, 'c');
string s8 = temp;

1.3 读写string 对象

int main()
{
    string str;
    cin >> str;
    cout << str << endl;

    return 0;
}

上面的程序定义了一个名为str的空string,然后将标准输入的内容读取到s中。在执行读取的操作时，string对象回自动忽略开头的空白(空格符, 换行符, 制表符等)并从第一个真正的字符开始读起,直到遇到下一个空白开始。

1.3.1 读取未知数量的string对象

int main()
{
    string word;
    while(cin >> word)
    {
        cout << word << endl;
    }
    return 0;
}

一旦遇到文件结束符或非法输入也就结束了。

1.3.2 getline()

getline读取一整行，直到遇到换行符，所读的内容存入string对象中。换行符也被读进来了,但是被丢弃了,并没有存储到string对象中

1.4 empty()和size()

empty函数根据string对象是否为空返回一个对应的布尔值。

size()函数的返回值是string::size_type类型,这是一个与机器无关的无符号整数类型。所以当出现string.size()就不要使用int了。

1.5 string的比较

关系运算符 <， <= ，>， >=都遵循以下的字典顺序：

  1. 如果两个string对象的长度不同,而且较短string对象的每个字符都与较长string对象对应位置上的字符相同,就说较短的string对象小于较长的string对象。
  2. 如果两个string对象在某些对应位置上不一致,则string对象比较的结果实际上是第一对相异字符比较的结果。

1.7.2 请说明string类的输入运算符和getline函数分别是如何处理空白字符的。

1.7.3 编写一段程序读取两个字符串，比较其是否相等并输出结果。如果不相等，输出比较大的那个字符串。改写上述程序，比较输入的两个字符串是否等长，如果不等长，输出长度较大的那个字符串。


void compareStrOne(const string &str1, const string &str2)
{
    if (str1 == str2)
    {
        cout << "The two strings are equals.\n";
    }
    else
    {
        cout << "The larger string is " << ((str1 > str2) ? str1 : str2) << ".\n";
    }
}

void compareStrTwo(const string &str1, const string &str2)
{
    if (str1.length() == str2.length())
    {
        cout << "The two strings are equal length.";
    }
    else
    {
        cout << "The longer string is " << ((str1.size() > str2.size()) ? str1 : str2) << ".\n";
    }

}

int main()
{
    string str1;
    string str2;
    while(cin >> str1 >> str2)
    {
        compareStrOne(str1, str2);
        compareStrTwo(str1, str2);
    }

    return 0;
}

1.9.3 分别用while循环和传统for循环重写第一题的程序，你觉得哪种形式更好呢？为什么?

string str = "some string";

// while循环
decltype(str.size()) index = 0;
while(index != str.size())
{
    if (!isspace(str[index]))
        str[index] = 'X';

    index++;
}

// 传统for循环
for (decltype(str.size()) index = 0;
     index != str.size() ; index++)
{
    if(isspace(str[index]))
        continue;
    str[index] = 'X';
}

我觉得各有优点，while循环判断条件更见简洁，传统for循环更倾向于将所有条件书写完再执行代码块，更容易理清楚思路。

2.3 添加元素

push_back()负责将一个元素添加到vector的尾端。

c++ 更习惯于先创建空的vector对象,在运行的时候动态添加元素。

vector<int> v2;

for(int i = 0; i != 100; ++i)

vector不能使用下标运算符去添加元素,但是可以用于访问已经存在的元素。

2.5 其他vector操作

v.empty() 如果v不含有任何元素，返回真，否则返回假
v.size() 返回v中元素的个数
v[n] 返回v中第n个元素位置的引用
v1 = v2 用v2中的元素拷贝替换v1中的元素
v1 = {a, b, c, …..} 用列表中的元素拷贝替换v1中的元素
v1 == v2 v1和v2相等当且仅当它们的元素数量相同且对应的元素值相同
v1 != v2
<, <=，>=, > 以字典顺序比较

vector对象的下标运算符可用于访问已经存在的元素，而不能用于添加元素。

void upperStr(string &str)
{
    /*
        you can use transform function. this is a standard libnary function.

    for (decltype(str.size()) index = 0; index < str.size(); index++)

}
int main()

}

2.6.3 读入一组整数并把它们存入一个vector对象，将每对相邻整数的和输出出来。改写你的程序，这次要求先输出第一个和最后一个元素的和，接着输出第2个和倒数第二2个元素的和，以此类推。

void outputOne(const vector<int> &v1)
{
    for (decltype(v1.size()) index = 0; index < v1.size() - 1; index++)
    {
        cout << v1[index] + v1[index + 1] << endl;
    }
}

void outputTwo(const vector<int> &v2)
{
    auto length = v2.size();
    for (decltype(v2.size()) index = 0; index < length / 2; index++)
    {
        auto result = v2[index] + v2[length - index - 1];
        cout << result << endl;
    }
}

int main()
{
    vector<int> vInt;
    int intBuffer;
    while (cin >> intBuffer)
    {
        vInt.push_back(intBuffer);
        if (cin.get() == '\n')
        {
            break;
        }
    }

    outputOne(vInt);        
    cout << "**********************" << endl;
    outputTwo(vInt);

    return 0;
}

2.7 迭代器

2.7.1 使用迭代器

/* 怎样获取迭代器？
    使用begin()和end().
    end成员负责返回指向容器尾元素的下一个元素的迭代器，也就是说，该迭代器指示的是容器的一个本不存在的“尾后”元素。
    这样的迭代器没什么实际含义，仅仅是个标记而已，表示我们已经处理完容器中所有元素。
    end() 返回的迭代器并不实际指示某个元素,所以不能对其进行递增或解引用的操作.
*/
auto b = v.begin();
auto e = v.end();

怎样称为一个有效的迭代器呢？

  • 或者指向某个元素,或者指向容器中尾元素的下一个位置

其次，但凡是使用了迭代器的循环体,都不要向使用迭代器所属的容器添加元素。因为会使迭代器失效。

2.7.2 迭代器运算符

ite*r 返回迭代器iter所选元素的引用
iter->mem 解引用iter并获取该元素名为mem的成员，等价于(*iter).mem
++iter 令iter指向容器中下一个元素
—iter 令iter指向容器中上一个元素
iter1 == iter2 判断两个迭代器是否相等(不相等)，如果两个迭代器指示的是同一个元素或者是同一个容器的尾后迭代器，则相等。反之，不想等。
iter1 != iter2 同上
